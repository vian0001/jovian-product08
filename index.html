Upgrade RouterOS melalui Command Line Interface (CLI)

Dari sumber ShellHacks (2023), upgrade lengkap pada perangkat MikroTik memerlukan peningkatan dua komponen: 1️⃣ MikroTik RouterOS sebagai sistem operasi (OS), dan 2️⃣ MikroTik RouterBOARD firmware sebagai bootloader. Untuk melakukan upgrade RouterOS melalui command line (terminal), jalankan perintah berikut:

[admin@MikroTik] > /system package update check-for-updates

Dari sumber MiViLiSNet (2020), jika Anda hanya mengelola beberapa perangkat MikroTik, Anda dapat memperbarui mereka satu per satu menggunakan tool WinBox. Ini adalah tugas yang sangat mudah jika Anda hanya menginstal versi long-term. Versi tersebut jarang diperbarui dan hanya berisi perbaikan bug utama. Jika Anda memiliki jumlah perangkat yang lebih besar yang berada di bawah kendali Anda, Anda mungkin menginginkan lebih banyak otomatisasi. Kabar baiknya adalah Anda dapat menggunakan keajaiban command line untuk melakukan tugas tersebut.

Dari sumber Forum MikroTik (2021), untuk melakukan upgrade RouterOS melalui CLI, gunakan perintah berikut:

/system package update check-for-updates;
/system package update install;

Dari sumber ShellHacks (2023), output dari perintah check-for-updates akan menampilkan informasi seperti berikut:

- sample output -
channel: stable
installed-version: 7.6
latest-version: 7.10.2
status: New version is available

Dari sumber MiViLiSNet (2020), proses update akan selesai setelah reboot. Selama proses shutdown, RouterOS akan memeriksa file .npk baru pada disk lokal dan menginstalnya. Untuk me-reboot router dari command line, ketikkan perintah:

[admin@MikroTik] > system reboot
Reboot, yes?
[y/N]: y
System will reboot shortly

Dari sumber MikroTik Documentation (2025), upgrading dari RouterOS v6 ke v7 dilakukan dengan cara yang sama seperti upgrading dalam rilis v6. Silahkan ikuti petunjuk upgrade untuk langkah-langkah yang lebih rinci.

Dari sumber MikroTik Documentation (2025), jika Anda saat ini menjalankan RouterOS versi 6 atau lebih lama, kami sarankan untuk terlebih dahulu upgrade ke rilis stable atau long-term terbaru di v6. Perhatikan bahwa di RouterOS v6.x Anda, hanya RouterOS rilis 7.12.1 yang akan ditampilkan sebagai rilis terbaru. Anda harus upgrade ke versi ini terlebih dahulu dan baru kemudian upgrade lebih lanjut ke versi yang lebih baru akan tersedia. Pada sebagian besar pengaturan RouterOS yang berjalan baik dengan versi v6 yang disebutkan di atas, tidak diperlukan langkah tambahan. Upgrading ke v7 akan secara otomatis mengonversi konfigurasi dan perangkat Anda akan berfungsi dengan segera.

Dari 3 pendapat tersebut dapat disimpulkan bahwa upgrade RouterOS melalui CLI dapat dilakukan dengan mudah menggunakan perintah /system package update check-for-updates untuk memeriksa pembaruan yang tersedia dan /system package update install untuk menginstal pembaruan tersebut, kemudian melakukan reboot perangkat. Proses upgrade dari versi v6 ke v7 memerlukan langkah tambahan dimana Anda harus terlebih dahulu upgrade ke versi 7.12.1.

Upgrade RouterBOOT Firmware melalui CLI

Dari sumber MikroTik Documentation (2025), sangat disarankan untuk mengupgrade bootloader setelah update RouterOS. Untuk mengupgrade bootloader, jalankan perintah "/system routerboard upgrade" di CLI, diikuti dengan reboot. Alternatifnya, navigasikan ke menu GUI System → RouterBOARD dan klik tombol "Upgrade", kemudian reboot perangkat.

Screenshot Proses Upgrade RouterBOOT
Gambar 3: Proses Upgrade RouterBOOT Firmware
Sumber: [Masukkan Sumber Gambar Disini]

Dari sumber ShellHacks (2023), setelah memperbarui MikroTik RouterOS, selalu disarankan untuk memperbarui firmware MikroTik RouterBOARD. Untuk memperbarui firmware MikroTik RouterBOARD menggunakan Winbox/Winfig, buka ⚙ "System" → "RouterBOARD" dan klik "Upgrade". Setelah upgrade firmware MikroTik RouterBOARD, reboot router agar perubahan diterapkan (⚙ "System" → "Reboot").

Dari sumber ShellHacks (2023), untuk upgrade firmware MikroTik RouterBOARD dari command-line (terminal), jalankan:

[admin@MikroTik] > /system routerboard print

Dari sumber MikroTik Documentation (2025), upgrade RouterBOOT biasanya mencakup perbaikan kecil pada operasi RouterBOARD secara keseluruhan. Disarankan untuk tetap mengupgrade versi ini. Jika Anda melihat bahwa nilai upgrade-firmware lebih besar dari firmware saat ini, Anda cukup melakukan perintah upgrade, menerimanya dengan y dan kemudian reboot dengan /system reboot:

[admin@mikrotik] /system routerboard> upgrade
Do you really want to upgrade firmware?
[y/n] y
echo: system,info,critical Firmware upgraded successfully, please reboot for changes to take effect!

Dari sumber MiViLiSNet (2020), setiap perangkat RouterBoard memiliki BIOS atau firmware sendiri. Firmware tersebut harus diupgrade setelah upgrade RouterOS. File firmware yang lebih baru dikemas di dalam file npk dan akan tersedia setelah upgrade RouterOS. Untuk melakukan upgrade firmware, gunakan perintah berikut:

[admin@MikroTik] > system routerboard upgrade
Do you really want to upgrade firmware?
[y/n] y
21:56:31 echo: system,info,critical Firmware upgraded successfully, please reboot for changes to take effect!
[admin@MikroTik] >

Dari sumber Forum MikroTik (2021), tidak mungkin untuk mengupgrade routerboot sebelum perangkat reboot ke versi ROS yang diupgrade. Satu-satunya jalan pintas yang dapat dilakukan adalah dengan menetapkan /system routerboard upgrade setelah upgrade ROS.

Dari 5 pendapat tersebut dapat disimpulkan bahwa upgrade RouterBOOT firmware melalui CLI dapat dilakukan dengan perintah /system routerboard upgrade setelah upgrade RouterOS dan diikuti dengan reboot perangkat. Penting untuk diperhatikan bahwa upgrade RouterBOOT harus dilakukan setelah upgrade RouterOS dan reboot pertama, tidak dapat dilakukan dalam satu langkah.

Otomatisasi Proses Upgrade dengan Script

Dari sumber MikroTik Documentation (2025), Anda dapat mengotomatisasi proses upgrade dengan menjalankan script di system scheduler. Script ini menanyakan server upgrade MikroTik untuk versi baru, jika respons yang diterima mengatakan "New version is available", script kemudian mengeluarkan perintah upgrade di bawah ini. Anda dapat mengotomatisasi proses upgrade dengan menjalankan script di system scheduler. Script ini menanyakan server upgrade MikroTik untuk versi baru, jika respons yang diterima mengatakan "New version is available", script kemudian mengeluarkan perintah upgrade.

Ilustrasi Automation Script MikroTik
Gambar 4: Otomatisasi dengan Script pada MikroTik

Dari sumber MikroTik Documentation (2025), berikut adalah contoh script untuk mengotomatisasi upgrade:

[admin@MikroTik] >/system package update check-for-updates once :delay 3s; :if ( [get status] = "New version is available") do={ install }

Dari sumber GitHub - spidermila/mikrotik_update (2025), ada tool command line yang mengotomatisasi update RouterOS pada armada perangkat Mikrotik. Tool ini akan terlebih dahulu melakukan backup perangkat dan mendownloadnya ke direktori lokal. Update dilakukan hanya jika backup dan download berhasil. Perilaku default tersebut dapat diganti dengan opsi -U atau -B. Tool ini menggunakan kunci ssh untuk mengautentikasi dengan perangkat Mikrotik.

Dari sumber GitHub - spidermila/mikrotik_update (2025), menggunakan script otomatisasi dapat mencegah terputusnya koneksi SSH saat melakukan update pada perangkat Mikrotik. Dengan menggunakan wrapper sederhana yang menjalankan script update dalam screen, proses update dapat terus berjalan meskipun koneksi terputus, dan Anda dapat kembali ke sesi tersebut ketika koneksi dipulihkan.

Dari 3 pendapat tersebut dapat disimpulkan bahwa proses upgrade RouterOS dan firmware dapat diotomatisasi menggunakan script baik di system scheduler MikroTik maupun tool eksternal. Script ini dapat memeriksa versi baru, melakukan backup konfigurasi, melakukan upgrade, dan me-reboot perangkat. Menggunakan screen atau tool serupa dapat mencegah terputusnya proses upgrade jika koneksi SSH terputus.

Memilih Channel Update yang Tepat

Dari sumber MikroTik Documentation (2025), versi RouterOS dirilis dalam beberapa "rantai rilis": Long term, Stable, Testing, dan Development. Saat mengupgrade RouterOS, Anda dapat memilih rantai rilis dari mana untuk menginstal paket baru. Long term: Dirilis jarang, dan hanya menyertakan perbaikan yang paling kritis, upgrade dalam satu cabang nomor tidak mengandung fitur baru.

Gambar 5: Channel Update pada RouterOS

Dari sumber MikroTik Wiki (2025), Current mencakup perbaikan yang sama ditambah peningkatan dan fitur baru. Setelah rilis current diuji selama beberapa bulan, rilis tersebut dipromosikan ke bugfix-only dan tidak lagi diperbarui dengan fitur.

Dari sumber MikroTik Wiki (2025), Release candidate dirilis beberapa kali per minggu. Mencakup fitur-fitur terbaru, dirilis tanpa pengujian intensif. Tidak direkomendasikan untuk produksi. Catatan: Sejak RouterOS v6.44beta6 channel rilis telah berganti nama - "bugfix" menjadi "long-term", "current" menjadi "stable" dan "release candidate" menjadi "testing".

Dari sumber MikroTik Wiki (2025), saat mengupgrade RouterOS, Anda dapat memilih rantai rilis dari mana untuk menginstal paket baru. Untuk instalasi mission critical, rantai rilis bugfixes-only disarankan, karena tidak menyertakan fitur baru yang baru ditambahkan dan disimpan untuk waktu yang lama di halaman download, dengan hanya perbaikan kritis yang diterapkan padanya. Versi bugfixed-only adalah rilis paling stabil tanpa fitur baru, hanya perbaikan paling penting. Diperbarui jarang, hanya ketika masalah kritis ditemukan dalam rilis bugfixes-only. Current mencakup perbaikan yang sama ditambah peningkatan dan fitur baru. Setelah rilis current diuji selama beberapa bulan, rilis tersebut dipromosikan ke bugfix-only dan tidak lagi diperbarui dengan fitur.

Dari 3 pendapat tersebut dapat disimpulkan bahwa MikroTik menawarkan beberapa channel update yang berbeda: Long-term (sebelumnya disebut bugfix), Stable (sebelumnya disebut current), dan Testing (sebelumnya disebut release candidate). Untuk lingkungan produksi dan sistem kritis, disarankan menggunakan channel Long-term yang lebih stabil dan hanya berisi perbaikan bug kritis. Channel Stable menyertakan fitur baru dan peningkatan, sementara channel Testing berisi fitur terbaru namun belum teruji secara intensif.

Prosedur Keamanan dalam Upgrade

Dari sumber ShellHacks (2023), sebelum memulai upgrade router MikroTik, pastikan Anda memiliki backup konfigurasi router Anda saat ini, karena ini penting jika terjadi kesalahan selama proses upgrade.

Dari sumber MikroTik Wiki (2025), Catatan: Jangan mencabut daya saat router sedang diupgrade.

Dari sumber Citraweb (2019), jika jaringan sudah stabil dengan menggunakan versi tertentu, akan lebih baik dipertahankan untuk menggunakan versi yang sama. Jangan terlalu sering upgrade/downgrade. Setiap versi memiliki kelebihan masing-masing dan terkadang hasilnya akan berbeda pada hardware yang berbeda. Biasakan membaca changelog. Hati-hati jika melakukan upgrade ke versi 6.20 atau yang lebih baru, khususnya pada Router yang menggunakan secondary storage.

Dari sumber MikroTik Documentation (2025), catatan penting, ini tidak akan berfungsi, jika Anda menjalankannya untuk pertama kali pada rilis yang lebih lama. Mungkin tidak melihat versi terbaru sebagai tersedia, jika Anda menjalankan v6.x, Anda harus terlebih dahulu memilih saluran "Upgrade" secara manual untuk melakukan upgrade rilis utama ke versi perantara v7.12.1, dan hanya setelah itu rilis v7 yang lebih baru akan terlihat dalam saluran upgrade.

Dari 4 pendapat tersebut dapat disimpulkan bahwa keamanan dalam proses upgrade meliputi: selalu membuat backup konfigurasi sebelum melakukan upgrade, tidak mencabut daya saat proses upgrade berlangsung, memahami changelog untuk mengetahui perubahan apa yang akan diterapkan, dan memperhatikan jalur upgrade yang benar (terutama saat upgrade versi mayor seperti dari v6 ke v7).

Downgrade RouterOS jika Diperlukan

Dari sumber Citraweb (2019), cara untuk melakukan downgrade cukup mudah. Seperti halnya proses upgrade, langkah pertama download terlebih dahulu modul RouterOS versi terdahulu yang diinginkan. Setelah selesai, "Copy & Paste" modul ke router menggunakan FTP. Selanjutnya, buka menu system -> packages maka akan muncul package list seperti pada gambar berikut ini: Klik Downgrade, lalu "yes" untuk perintah reboot. Setelah Router menyala kembali, sudah akan menggunakan versi RouterOS yang diinginkan.

Dari sumber MikroTik Wiki (2025), juga dimungkinkan untuk menggunakan backup booter dengan menyalakan perangkat, dengan tombol RESET ditekan. Terkadang RouterBOOT menerima upgrade firmware (lihat Changelog). Hanya mungkin untuk mengupgrade RouterBOOT utama, jadi jika terjadi kegagalan, Anda dapat menggunakan backup booter untuk memulai perangkat dan menurunkan versi loader utama.

Dari sumber MikroTik Wiki (2025), perintah ini menunjukkan versi RouterBOOT saat ini dari perangkat Anda, dan upgrade yang tersedia yang termasuk dalam paket routerboard.npk, atau jika Anda mengunggah file FWF yang sesuai dengan model perangkat. Catatan: Downgrade juga dimungkinkan dengan mengunggah file *.FWF dengan versi yang lebih lama.

Dari 3 pendapat tersebut dapat disimpulkan bahwa jika diperlukan, RouterOS dapat di-downgrade ke versi sebelumnya dengan cara mendownload modul RouterOS versi yang diinginkan, meng-copy ke router, dan menggunakan opsi Downgrade di menu system -> packages. Untuk RouterBOOT, downgrade dapat dilakukan dengan mengunggah file FWF dengan versi yang lebih lama atau menggunakan backup booter dengan menekan tombol RESET saat menyalakan perangkat.
